# Contributing

Thanks for helping improve SnapFrame.

## Getting Started

- Use AutoHotkey v2 for all script changes.
- Keep the tool focused on fixed-size screenshot workflows.
- Prefer small, testable changes.

## Development Notes

- Main script: `snapframe.ahk`
- Local planning and personal notes can live in `docs/`, but that folder is not tracked by git.
- Generated screenshots and compiled executables should not be committed.

## Pull Requests

- Describe the problem and the behavior change clearly.
- Include manual test steps for screenshot flow changes.
- Keep unrelated refactors out of the same PR when possible.
